# corndogs clients

Generated from [`../csil/corndogs.csil`](../csil/corndogs.csil) by
[`../csil/generate.sh`](../csil/generate.sh) (csilgen, pinned rev in that script).
The wire is **CBOR over HTTP**: each operation is
`POST {baseUrl}/v1alpha1/{service}/{method}` with a CBOR-encoded request body and
a CBOR-encoded response (`service` = lowercased service name `corndogs`; `method`
= the operation name as written in CSIL, e.g. `SubmitTask`). Map/field keys on the
wire are the CSIL field names verbatim (snake_case). The canonical contract is
documented normatively in csilgen's `docs/cbor-wire-contract.md`.

## Layout

```
clients/<lang>/gen/   <- generated by csilgen; wholly owned, wiped+rewritten each run. DO NOT EDIT.
clients/<lang>/       <- hand-written support (CBOR transport, package manifest) importing from ./gen/
```

The `csil-gen-check` reactorcide job re-runs `generate.sh` and fails if the
committed `gen/` output drifts. Edit the CSIL and regenerate; never edit `gen/`.

## Status per language

All four languages now generate a real, typed, transport-agnostic client (csilgen
client-generator requests resolved 2026-06-08). Each generated client delegates
the wire to a small caller-supplied `Transport`; the hand-written part is a
~15–30 line CBOR-over-HTTP implementation of that interface.

| Language | Generated client | CBOR transport (hand-written) |
|---|---|---|
| **typescript** | `gen/client.gen.ts`, `gen/types.gen.ts` | ✅ `transport.ts` (`CborHttpTransport`, cbor-x) + `index.ts` + `package.json` |
| **go** | `gen/client.gen.go`, `gen/types.gen.go` | ✅ `transport.go` (`CBORTransport`, net/http + fxamacker/cbor) + `go.mod` |
| **rust** | `gen/client.rs`, `gen/types.rs`, `gen/mod.rs` | ✅ `transport.rs` (`CborHttpTransport`, reqwest + ciborium) + `lib.rs` + `Cargo.toml` |
| **python** | `gen/client.py`, `gen/types.py` | ✅ `transport.py` (`CborHttpTransport`/`connect`, urllib + cbor2) + `__init__.py` + `pyproject.toml` |

All four clients are complete and typed (Go `(Resp, error)`, Rust
`Result<Resp, ClientError>`, Python dataclasses + `ServiceError`, TS Promises) and
each has a CBOR-over-HTTP transport. Verified: Go `go build`, Rust `cargo check`,
Python transport logic (`_to_plain`/`_from_data`/introspection).

> The Python transport reconstructs typed responses by introspecting the client's
> return annotations, because the generated `Transport.call` isn't handed the
> response type (Go/Rust are). Tracked in csilgen
> `docs/csilgen-requests/2026-06-08-python-transport-response-type.md`; once fixed,
> that introspection goes away.
